# 视频防抖技术调研报告（2015–2026）

> 调研目标：梳理近十年视频防抖（Video Stabilization）技术进展，以**效果好、鲁棒性高**为目标，以**工程落地**为导向，给出技术路线对比与选型建议。
> 撰写日期：2026-07

---

## 1. TL;DR

- **学术界**近十年经历了三次范式迁移：传统 2D 特征轨迹 + 路径平滑（2013–2016）→ 深度学习端到端 / 无监督（2018–2021）→ 3D 感知与生成式全帧合成（2021–2025）。
- **工业界落地的主流方案与学术前沿并不重合**：手机、运动相机、无人机上真正大规模落地的是**陀螺仪驱动的 EIS（+OIS 融合）**，而不是纯视觉深度学习方法。传感器方案在鲁棒性、功耗、延迟上仍有压倒性优势。
- **落地选型的第一个问题是：能不能拿到陀螺仪数据？**
  - 能 → 走 gyro-based EIS 路线（参考 Gyroflow / Google Fused Stabilization），效果和鲁棒性最好，成本最低；
  - 不能（存量视频、第三方视频、后期处理）→ 走纯视觉路线，离线场景推荐"稳健 2D 网格路径平滑 + 深度学习全帧补全"的混合方案，在线场景推荐轻量光流 + 学习式路径平滑。
- 纯视觉 SOTA（FuSta、RStab、GaVS 等）画质上限高，但**鲁棒性与速度尚不适合直接产品化**，适合作为离线高质量档位或云端处理。

## 2. 技术发展脉络

### 2.1 传统 2D 方法（~2016 前，至今仍是工程基石）

核心流程：**运动估计（特征跟踪/光流）→ 相机路径平滑 → 网格形变渲染 + 裁剪**。

| 方法 | 年份/出处 | 要点 |
|---|---|---|
| L1 Optimal Camera Paths | 2011, CVPR（YouTube 防抖） | 用 L1 优化生成由恒定、线性、抛物线段组成的"电影感"路径，工程上极其经典 |
| Bundled Camera Paths | 2013, SIGGRAPH | 网格划分多路径，处理视差；奠定"网格路径"框架 |
| SteadyFlow | 2014, CVPR | 用平滑化的稠密光流（SteadyFlow）替代特征轨迹，规避长轨迹断裂问题 |
| MeshFlow | 2016, ECCV | 稀疏特征投票到网格顶点的"轻量稠密流"，**可在线运行**，是很多嵌入式实现的原型 |

**工程评价**：这一代方法可解释、可调参、CPU 可实时，至今仍是 ffmpeg vidstab、OpenCV videostab、Adobe Warp Stabilizer 等落地产品的骨架。弱点：大视差、大动态前景、低纹理/低光下特征跟踪失败会导致明显失效，且必须裁剪（通常损失 10–20% 视野）。

### 2.2 深度学习方法（2018–2021）

| 方法 | 年份/出处 | 要点 |
|---|---|---|
| StabNet | 2018, TIP | 首个监督 CNN 方案，配套 **DeepStab** 数据集（手持/稳定器同步拍摄对） |
| DIFRINT | 2020, TOG | 迭代帧插值实现**全帧**（无裁剪）防抖，开创全帧路线 |
| Yu & Ramamoorthi | 2020, CVPR | 直接在光流域优化 warp field，逐视频优化，画质好但慢 |
| DUT | 2020 | 无监督 Deep Unsupervised Trajectory 方法，摆脱成对数据依赖 |
| Deep3D Stabilizer | 2021, CVPR | 逐视频估计深度+位姿，3D 重投影防抖，开启 3D 感知路线 |
| FuSta (Hybrid Neural Fusion) | 2021, ICCV | 多帧特征融合合成全帧结果，长期占据画质 SOTA，但速度慢、显存大 |

**工程评价**：监督方案受 DeepStab 数据集规模/场景限制，泛化一般；无监督与逐视频优化方案画质好但多为**离线、秒级/分钟级每帧**的开销。这一代的最大遗产是"全帧化"（不裁剪）和无监督训练范式。

### 2.3 传感器融合方法（2017 至今，工业落地主线）

| 方法/产品 | 年份 | 要点 |
|---|---|---|
| [Google Fused Video Stabilization](https://research.googleblog.com/2017/11/fused-video-stabilization-on-pixel-2.html) | 2017, Pixel 2 | OIS + 陀螺仪 EIS 联合：gyro 估计帧内逐行位姿（同时校正果冻效应），虚拟相机路径前瞻平滑，手机端实时 |
| [Deep Online Fused Video Stabilization](https://arxiv.org/abs/2102.01279) | 2022, WACV | 陀螺仪 + 光流联合输入 LSTM 预测虚拟相机位姿，无监督训练，仅 10 帧延迟，代表"传感器 + 学习"融合方向 |
| [Gyroflow](https://github.com/gyroflow/gyroflow)（开源） | 2021– | 读取相机内录 gyro 数据做后期防抖：gyro/视频时间同步 → 3D 旋转路径重建 → 平滑 → 含果冻校正与镜头畸变校正的逐像素重投影。支持 GoPro/DJI/Sony/Insta360 等 |
| From Gyroscope to Pixels | 2025 | 物理先验 + 学习的在线 gyro 防抖，学术界回归传感器路线的代表 |

**工程评价**：陀螺仪给出的 3D 旋转是**厘米级视差以外抖动的真值**，不依赖图像内容，因此在低光、快速运动、大动态前景、纹理缺失等纯视觉方法的全部失效场景下都稳定工作，且计算量极小（只需路径平滑 + 一次网格重投影）。**这是"鲁棒性高、可落地"这两个目标下的最优路线**。局限：只能校正旋转抖动（平移抖动引起的视差无法用 gyro 校正，但手持场景旋转占主导）；需要硬件/固件配合记录 gyro 流并精确同步时间戳。

### 2.4 3D 感知与生成式方法（2021–2025，画质前沿）

| 方法 | 年份/出处 | 要点 |
|---|---|---|
| Fast Full-frame via Iterative Optimization | 2023, ICCV | 迭代优化式全帧防抖，比 FuSta 快一个量级 |
| Minimum Latency Online | 2023, ICCV | 面向在线场景的最小延迟深度防抖 |
| [RStab: 3D Multi-frame Fusion](https://openaccess.thecvf.com/content/CVPR2024/html/Peng_3D_Multi-frame_Fusion_for_Video_Stabilization_CVPR_2024_paper.html) | 2024, CVPR | 体渲染式多帧 3D 融合（Stabilized Rendering），全帧且保结构 |
| [GaVS](https://sinoyou.github.io/gavs/) | 2025, SIGGRAPH | 把防抖重构为"局部 3DGS 重建 + 平滑轨迹重渲染"，全帧、几何一致性最好；但依赖 GLOMAP 视觉里程计，**低光/自拍等场景位姿估计会失败**，需逐视频 test-time 微调 |
| StabStitch++ | 2025 | 多摄像头视频拼接 + 防抖联合的无监督框架 |
| 无监督在线防抖（No Labels, No Look-Ahead） | 2026 | 经典先验 + 无监督学习的在线方案，方向是把传统方法的可靠性带回学习框架 |

**工程评价**：3D/生成式路线代表画质上限——真正的全帧、无畸变、几何一致。但共同短板是：依赖位姿/深度估计（本身是脆弱环节）、需要 GPU、多为离线甚至逐视频优化。GaVS 作者自己承认在低光和少静态特征场景失效。**短期内适合云端离线高质量档，不适合端侧实时**。

### 2.5 评测体系

- **经典三指标**（NUS 基准，Liu et al.）：Cropping Ratio（裁剪率）、Distortion（畸变，全局单应各向异性）、Stability（稳定度，路径频域低频能量占比）。
- 新趋势：几何一致性指标（GaVS 提出）、无参考质量评估模型 [StableVQA](https://arxiv.org/abs/2308.04904)（2023）、用户主观实验。
- 常用数据集：NUS、DeepStab、Selfie Video、GaVS 数据集。
- **落地提醒**：三个经典指标可被"过度平滑 + 大裁剪"游戏化，产品评测务必加入主观盲评和失效率（fallback 触发率）统计。

## 3. 技术路线横向对比（落地视角）

| 路线 | 代表 | 画质上限 | 鲁棒性 | 计算/延迟 | 是否裁剪 | 落地成熟度 |
|---|---|---|---|---|---|---|
| 传统 2D 网格 | MeshFlow / Bundled Paths / Warp Stabilizer | 中 | 中（低光/大前景失效） | CPU 实时可在线 | 裁剪 10–20% | ★★★★★ 已大规模落地 |
| 陀螺仪 EIS（±OIS） | Pixel Fused / GoPro HyperSmooth / DJI RockSteady / Gyroflow | 中高 | **最高**（与图像内容无关） | 极低，端侧实时 | 裁剪（可调） | ★★★★★ 行业标配 |
| Gyro + 学习融合 | Deep Online Fused (WACV'22) | 高 | 高 | 低延迟（~10 帧） | 裁剪小 | ★★★☆ 手机厂商在用类似方案 |
| 深度学习 2D（无监督） | DUT / Yu&Ramamoorthi | 高 | 中 | GPU，多为离线 | 部分裁剪 | ★★☆ |
| 全帧合成 | DIFRINT / FuSta / Fast Full-frame'23 | 很高 | 中低（合成伪影风险） | GPU 重，离线 | **无裁剪** | ★★ 云端/后期工具 |
| 3D 重建渲染 | Deep3D / RStab / GaVS | **最高** | 低（依赖位姿/深度估计） | 很重，常需逐视频优化 | 无裁剪 | ★ 研究阶段 |

## 4. 工业界落地现状

- **手机**：OIS（镜头级光学防抖）+ gyro EIS 融合已是旗舰机标配（Google Fused Stabilization 是公开细节最全的参考实现），近年叠加了逐行果冻校正、运动模糊感知的路径规划、前瞻缓冲（增加 <1s 延迟换取更平滑路径）。
- **运动相机/无人机**：GoPro HyperSmooth、DJI RockSteady/HorizonSteady、Insta360 FlowState 全部是 gyro 驱动 EIS；广角 + 大裁剪余量是它们效果好的关键前提。多数机型同时**内录 gyro 数据**供后期二次处理。
- **后期软件**：Adobe Warp Stabilizer（子空间/网格路线）仍是纯视觉后期事实标准；[Gyroflow](https://gyroflow.xyz/) 作为开源 gyro 后期方案已成 FPV/运动相机社区标配，且可叠加在 HyperSmooth 之上。
- **开源基础设施**：ffmpeg `vidstab`、OpenCV videostab（传统 2D）；深度学习侧 FuSta、DUT、RStab、GaVS 均有官方代码（见 [awesome-video-stabilization](https://github.com/yaochih/awesome-video-stabilization)），但均无生产级工程化。

## 5. 选型建议（以效果 + 鲁棒性 + 落地为准绳）

### 场景 A：自有硬件，可拿到陀螺仪数据（相机、无人机、机器人、可穿戴）
**首选 gyro EIS 路线**，参考 Gyroflow（后期）/ Google Fused（实时）的公开设计：
1. gyro-视频时间戳精确同步（亚毫秒级，最关键的工程环节）；
2. 四元数旋转路径重建 → 前瞻窗口平滑（L1 或低通 + 边界约束防止越界）；
3. 逐行（rolling shutter）重投影 + 镜头畸变校正一次性合成。

效果稳定、失效模式几乎为零、算力可忽略。若追求更小裁剪与平移抖动抑制，可加轻量光流做残差校正（Deep Online Fused 的思路）。

### 场景 B：处理存量/第三方视频，离线（云端剪辑、素材增强）
**推荐两档**：
- 标准档：稳健 2D 路线（特征 + 光流混合运动估计、RANSAC 剔除前景、L1/网格路径平滑），失败可控、速度快；
- 高质量档：在标准档之上叠加全帧补全（Fast Full-frame ICCV'23 是速度/质量平衡点；FuSta/RStab 画质更高但更慢），并设置**质量守门**——合成伪影检测不通过时回退到标准档结果。

### 场景 C：无传感器的实时/在线（视频会议、直播、旧设备）
轻量光流（如 RAFT-small 级别或传统金字塔 LK）+ 学习式在线路径平滑（Minimum Latency ICCV'23、2026 无监督在线方法的方向），控制在 0–10 帧延迟；接受裁剪，重点做**失效检测 + 平滑降级**（检测到运动估计不可靠时逐渐衰减防抖强度而不是跳变）。

### 通用工程要点（鲁棒性的真正来源）
1. **失效检测与优雅降级**是产品防抖和论文防抖的最大差别：跟踪内点率、单应残差、光流置信度任一异常时衰减平滑强度或直通输出；
2. **裁剪率预算化**：把最大裁剪率作为硬约束进入路径优化（L1 路线天然支持），而不是事后裁剪；
3. **果冻效应校正**必须与防抖一起做（gyro 逐行位姿或视觉逐行单应），否则强防抖会放大果冻感；
4. 大动态前景（行人、车辆）场景：语义/运动分割掩掉前景再估计背景运动，是传统与深度方案共同的有效补丁；
5. 评测用 NUS 三指标 + 主观盲评 + 分场景失效率（低光/夜景、行走、跑动、车载、大前景五类至少各覆盖）。

## 6. 挑战与趋势展望

1. **传感器 + 学习融合是最有落地价值的方向**：gyro 提供鲁棒骨架，网络只学残差（视差、平移抖动、路径风格），兼得鲁棒性与画质，学术界 2022 年后明显回流此方向。
2. **生成式全帧合成**（扩散模型视频 outpainting 与防抖的结合）会消灭裁剪问题，但幻觉内容的可控性与时序一致性未解决，2–3 年内定位仍是离线增强。
3. **3D 表征（3DGS）路线**（GaVS 为代表）画质与几何一致性最好，瓶颈在鲁棒的实时位姿估计——这本质上把防抖问题转嫁给了 SLAM/VO，短期不改变落地格局。
4. **评测正规化**：StableVQA 等无参考指标 + 几何一致性指标正在替代易被游戏化的经典三指标。
5. 端侧算力增长会让"轻量网络在线防抖"逐步吃掉传统 2D 的份额，但**gyro EIS 在有传感器的设备上地位十年内难以撼动**。
