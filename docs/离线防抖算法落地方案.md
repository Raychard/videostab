# 离线视频防抖算法落地方案（DUT 框架 × NLNL 改进）

> 目标：以 DUT 的无监督网格轨迹框架为骨架，融合 NLNL 的鲁棒性改进，构建一个**离线、纯视觉、可产品化**的防抖算法。
> 定位：处理存量/第三方视频（无陀螺仪数据），部署形态为服务端 GPU 批处理或桌面端后期工具。
> 配套文档：[技术调研报告](./视频防抖技术调研报告.md) · [DUT/NLNL 对比报告](./DUT算法详解与NoLabels-NoLookAhead对比.md)
> 撰写日期：2026-07

---

## 1. 设计总纲

**核心决策：离线场景下，NLNL 的"因果化"改动不要，"鲁棒化"改动全要。**

NLNL 相对 DUT 的改进分两类：
- **为在线性付出的妥协**（因果核、零前瞻、只看过去 7 帧）——离线场景**不采纳**。双向全局平滑是离线方法相对在线方法的最大画质红利，必须保留并强化；
- **为鲁棒性做的升级**（多检测器协同前端、关键点均匀化、软融合多单应、运动自适应/频域损失、轻量化模块）——**全部采纳**。这些正好补上 DUT 的产品化短板。

在此之上，补三层论文不管但产品必需的工程：**失效检测与优雅降级、镜头切分与裁剪预算、吞吐优化**。

```
                    ┌─ 论文不覆盖、产品必需 ─────────────────────────────┐
输入视频 ─► 预处理  │  转场切分 / 元数据 / 降采样代理流                    │
            │       └──────────────────────────────────────────────────┘
            ▼
   ┌─ Stage 1 运动估计（NLNL 前端）────────────────────────────┐
   │ 多检测器协同(ALIKE+ORB) + 空间均匀化 + RAFT 光流稀疏引导融合 │──► 质量守门 QG-1
   └──────────────────────────────────────────────────────────┘
            ▼
   ┌─ Stage 2 运动传播（DUT 框架 + NLNL 软融合）────────────────┐
   │ K类软融合多单应先验 + 轻量残差网络(Ghost+ECA, ~25K 参数)     │──► 质量守门 QG-2
   └──────────────────────────────────────────────────────────┘
            ▼
   ┌─ Stage 3 轨迹平滑（DUT 双向 + NLNL 损失强化）──────────────┐
   │ 全局双向动态核平滑 + 运动自适应二阶惩罚 + 频域抑制            │
   │ + 裁剪预算硬约束                                            │
   └──────────────────────────────────────────────────────────┘
            ▼
   ┌─ Stage 4 渲染 ────────────────────────────────────────────┐
   │ 标准档：网格 warp + 预算内裁剪                               │
   │ 高质档：+ 全帧补全（可选，二期）                             │
   └──────────────────────────────────────────────────────────┘
```

## 2. 各模块详细设计

### 2.1 预处理层

| 项 | 方案 | 理由 |
|---|---|---|
| 转场切分 | PySceneDetect（内容检测模式）按镜头切段，各段独立防抖 | 跨转场平滑轨迹是经典失效源，DUT/NLNL 均未处理 |
| 代理分辨率 | 运动估计统一在 **480p 代理流**上做，warp 在原分辨率执行 | DUT 原文即 480p 评测；网格运动场与分辨率无关，可等比放大；吞吐提升 4–10 倍 |
| 元数据 | 保留原始帧率/时间戳/旋转 flag，warp 后原样回写 | 变帧率(VFR)视频是常见线上 case，时间戳错误会导致平滑核失配 |
| VFR 处理 | 按时间戳归一化轨迹采样间隔，而非按帧序号 | 同上 |

### 2.2 Stage 1：运动估计（采纳 NLNL 前端，离线化选型）

**关键点：双检测器协同，不做四个。**
- 主检测器 **ALIKE-T**（轻量、可微、弱纹理表现好）+ 备份检测器 **ORB**（零学习成本、极端场景兜底、CPU 兜底路径可用）；
- NLNL 用四个检测器（SIFT/ORB/SuperPoint/ALIKE）是为论文完备性，产品上两个已覆盖主要互补性，多一倍前端成本收益递减；
- 加权 NMS 合并 + **空间选择性聚类均匀化**（照抄 NLNL：网格化图像，每格保留 top-k 响应点），目标 512–1024 点/帧。

**光流：RAFT-S（离线可用非因果流，不必用 MemFlow）。**
- 相邻帧双向光流，**前后向一致性校验**产出逐像素置信度图（这是离线的又一红利：NLNL 因果设定拿不到后向流）；
- 稀疏引导融合（NLNL 做法）：关键点邻域内信任光流，邻域外用关键点运动插值，按置信度加权。

**动态前景剔除（DUT/NLNL 共同短板的补丁，标准档必做）：**
- 第一版：RANSAC 多单应内点判定 + 前后向流一致性双重过滤，剔除运动异常关键点（纯几何，零额外模型）；
- 第二版（可选增强）：接入轻量分割（如 RTMDet/FastSAM 的人车类 mask）在人流/车流密集场景屏蔽前景关键点。以 badcase 率决定是否上。

**质量守门 QG-1**（进入降级逻辑的信号，见 §3）：有效关键点数、内点率、光流前后向一致率。

### 2.3 Stage 2：运动传播（DUT 骨架 + NLNL 软融合）

- **多单应初始化**：K-means 对关键点运动聚类，K 自适应（BIC 或轮廓系数在 1–3 中选，替代 DUT 固定 2 类 + 手调 20% 阈值——这是 DUT 自述需逐场景调参的点）；逐类 RANSAC 估单应；
- **软融合**（NLNL 做法替换 DUT 硬归属）：每个网格顶点对 K 个单应按邻域关键点类别分布 + 距离衰减做加权混合，消除平面边界处的撕裂；
- **残差细化网络**：照搬 NLNL 的 EfficientMotionPro 设计（Ghost 模块 + ECA 注意力，~25K 参数），输入稀疏关键点运动 + 位置编码，输出逐网格顶点残差；
- 网格分辨率 16×9（1080p 下约 120px/格），与 DUT 一致量级。

**质量守门 QG-2**：网格单应重投影残差、保形约束违反量。

### 2.4 Stage 3：轨迹平滑（本方案的画质核心，离线红利全在这）

保留并强化 DUT 的双向平滑：
- **全局双向动态核**：DUT 的 3D CNN 动态核预测器，但时间感受野从局部窗扩到 **±30 帧**（1–2 秒），快速运镜/抖动的区分更准；
- **损失强化（移植 NLNL）**：
  - 运动自适应二阶惩罚（大幅运镜段自动降低平滑权重 → 保跟拍意图，防"过稳"）；
  - DFT 频域高频抑制项（对残余高频抖动比纯时域差分更敏感）；
- **裁剪预算硬约束（产品项，两篇论文都没有）**：把最大裁剪率 c_max（默认 12%，产品可调档：8%/12%/18%）作为轨迹优化的硬边界——平滑轨迹使任一网格顶点越出预算框时投影回边界，代替 DUT 软性的"距离保持"损失。用户拿到的裁剪是**确定性承诺**而不是期望值；
- 迭代求解器保留 DUT 方案（15 次迭代收敛），离线无延迟压力。

### 2.5 Stage 4：渲染

- 标准档：平滑前后网格差 → 逐网格单应 → 原分辨率反向 warp（GPU bilinear/bicubic）→ 按预算统一裁剪 + 等比放大回原分辨率；
- **果冻效应**：第一版不做显式逐行校正（纯视觉逐行估计成本高、收益场景窄），但网格 warp 天然吸收部分果冻形变；若车载/快门速度慢的 badcase 占比高，二期加基于网格行间插值的近似校正；
- 高质档（二期）：裁剪区域用全帧补全替代（首选 Fast Full-frame ICCV'23 的迭代优化方案，速度/质量平衡最好），**必须带合成伪影检测守门**，不通过则回退标准档结果。

## 3. 失效检测与优雅降级（产品鲁棒性的真正来源）

三级降级链，任何一级触发都**平滑过渡**（防抖强度在 1 秒内线性衰减，绝不跳变）：

| 级别 | 触发条件（QG 信号） | 行为 |
|---|---|---|
| L0 完整管线 | 默认 | 多单应 + 残差网络 + 动态核平滑 |
| L1 保守模式 | 内点率 < 50% 或有效关键点 < 128 或网格残差超阈值 | 退化为单全局单应 + 固定高斯核平滑（≈经典方法，几乎不会产生畸变） |
| L2 直通 | 关键点 < 32 或单应估计不收敛（黑场、纯天空、剧烈模糊） | 原样输出该段，日志记录 |

- 触发统计上报为核心质量指标：**分场景 fallback 率**是版本迭代的北极星指标之一；
- 段级恢复：L1/L2 按镜头段生效，下一段重新尝试 L0。

## 4. 训练方案

- **范式**：完全沿用 DUT 无监督配方（不需要任何成对数据），损失 = 关键点/顶点一致性 + 单应投影一致性 + 网格保形 + {自适应二阶时域 + 频域} 平滑项；
- **数据**：自采 + 公开混合，**按目标场景配比**——手持行走/跑动 30%、车载 20%、无人机 20%、室内低光 15%、大动态前景（街拍人流）15%；只需不稳定视频，采集成本低，目标 ≥50 小时；
- **训练开销参考**（NLNL 实测）：传播网络 ~12h、平滑网络 ~3h（单张 RTX 4090）——迭代成本很低，支持每周级重训；
- **课程策略**：先在常规手持数据上训练收敛，再混入低光/大前景难例微调，避免难例主导早期训练；
- 前端（ALIKE/RAFT）用公开预训练权重冻结，**只训练传播残差网络和平滑核网络**（合计 <50K 参数）——这是本方案训练风险低的根本原因。

## 5. 评测与验收

### 5.1 指标体系

| 类别 | 指标 | 验收线（v1.0） |
|---|---|---|
| 客观 | NUS 三指标 C/D/S（自建统一评测框架重跑所有基线） | S ≥ DUT 复评值，D ≥ 0.93，C ≥ 0.88（预算 12% 档） |
| 客观 | StableVQA 无参考分 | ≥ DUT 基线 |
| 主观 | 20 人盲评（成对比较 vs DUT / vs Adobe Warp Stabilizer） | 胜率 ≥ 55% vs DUT |
| 鲁棒性 | 分场景 fallback 率（五类场景各 ≥30 段回归集） | L2 直通率 < 2%，L1 率 < 10% |
| 性能 | 吞吐（1080p30，A10/4090 单卡） | ≥ 3× 实时（即 ≥90 FPS 等效） |

### 5.2 回归测试集

自建 150+ 段分场景难例库（低光/夜景、行走跑动、车载、大前景、快速运镜各 ≥30 段），每次版本迭代全量跑：客观指标 + fallback 率 + 抽样人工盲评。**badcase 只进不出**——修复后保留在集内防回归。

## 6. 工程实现

### 6.1 技术栈与部署

- PyTorch 训练；推理路径 TensorRT（服务端）/ ONNX Runtime（桌面端）导出——本方案自训模块均为小型 CNN，导出无算子风险；ALIKE/RAFT 均有成熟 TRT 方案；
- **三阶段进程内流水线**（NLNL 的多线程思想用于离线吞吐）：运动估计（GPU 重）/ 传播+平滑（轻）/ 渲染 IO（编解码重）三个 worker 池经队列衔接，编解码用 NVDEC/NVENC；
- 显存预算：480p 代理估计 + 小网络，单路 < 4GB，单卡可并发多路。

### 6.2 仓库结构建议

```
videostab/
├── preprocess/        # 转场切分、代理流、VFR 归一化
├── motion/            # Stage1: 检测器协同、均匀化、RAFT 融合、前景剔除
├── propagation/       # Stage2: 软融合多单应 + 残差网络
├── smoothing/         # Stage3: 双向动态核 + 裁剪预算求解器
├── render/            # Stage4: 网格 warp、裁剪、（二期）全帧补全
├── guard/             # QG 信号、三级降级状态机
├── train/             # 无监督训练、数据管道
├── eval/              # 统一评测框架（C/D/S、StableVQA、fallback 统计）
└── pipeline.py        # 流水线编排
```

### 6.3 里程碑

| 阶段 | 内容 | 出口标准 | 预估 |
|---|---|---|---|
| **M0 基线** | 跑通 DUTCode 官方代码，搭建统一评测框架，重跑 DUT/MeshFlow 基线 | 评测框架产出可复现的 C/D/S 报表 | 2 周 |
| **M1 前端替换** | ALIKE+ORB 协同 + 均匀化 + RAFT 融合 + 几何法前景剔除，接入 DUT 后端 | 低光/大前景难例集上 D 指标与失效率显著改善 | 3–4 周 |
| **M2 传播升级** | 软融合多单应 + 残差网络重训 | 视差难例 D ≥ DUT；消除平面边界撕裂 badcase | 3 周 |
| **M3 平滑升级** | ±30 帧双向动态核 + 自适应/频域损失 + 裁剪预算硬约束 | S 超 DUT；裁剪率 100% 满足预算承诺 | 3 周 |
| **M4 产品化** | 降级状态机、转场切分、流水线吞吐优化、TRT 导出 | §5.1 全部验收线达标 | 4 周 |
| **M5 二期** | 全帧补全高质档、近似果冻校正、分割法前景剔除 | 按 badcase 数据决定优先级 | — |

关键依赖：M1 是最大的效果杠杆（DUT 的失效根源在前端）；M4 的降级状态机是能不能"敢上线"的分水岭，不可砍。

## 7. 风险与对策

| 风险 | 概率 | 对策 |
|---|---|---|
| 前端替换后 DUT 预训练权重失配，重训不收敛 | 中 | M1 先冻结后端只换前端做 A/B；重训从 DUT 权重 warm-start；自训参数 <50K，试错成本低 |
| 大动态前景仍是 badcase 主源 | 高 | 已列 M5 分割方案；短期靠 L1 降级兜底保"不出错" |
| C/D/S 达标但主观不佳（指标可被过平滑游戏化） | 中 | 主观盲评进验收线；自适应二阶惩罚天然抑制过平滑 |
| 全帧补全伪影引发用户投诉 | 中 | 高质档单独开关 + 伪影守门 + 默认关闭 |
| 吞吐不达标 | 低 | 代理分辨率可再降（384p）；瓶颈在 RAFT，可换 NeuFlow v2 级轻量流 |

## 8. 方案与两篇论文的差异清单（速查）

| 设计点 | DUT | NLNL | 本方案 |
|---|---|---|---|
| 平滑方向 | 双向（离线） | 因果（在线） | **双向 ±30 帧**（离线红利，强化） |
| 关键点 | RFNet | 4 检测器协同 | **ALIKE-T + ORB 双检测器**（收益/成本折中） |
| 光流 | PWC-Net | MemFlow（因果） | **RAFT-S + 前后向一致性置信度**（离线可用双向） |
| 多单应 | 固定 2 类硬归属 | K 类软融合 | **自适应 K + 软融合** |
| 平滑损失 | 时域差分 | + 自适应二阶 + 频域 | 同 NLNL |
| 裁剪控制 | 软损失项 | 软损失项 | **硬预算约束（产品承诺）** |
| 前景处理 | 无 | 无 | **几何剔除（一期）+ 分割（二期）** |
| 失效降级 | 无 | 无 | **三级降级状态机** |
| 转场/VFR | 无 | 无 | **预处理层处理** |
| 全帧 | 无 | 无 | 二期高质档（带守门） |
