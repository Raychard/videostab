## 1. TL;DR

- **DUT（2020/2022）是无监督网格轨迹防抖的开山之作**：首次证明只用不稳定视频（无需稳定/不稳定成对数据）就能训练防抖网络，把经典"关键点 → 网格轨迹 → 平滑"管线中每一步换成可微模块，用空间/时间一致性作为自监督信号。
- **NLNL（2026）可以看作 DUT 思想的"在线化 + 工程化"直系后继**：同样是无监督、同样是网格轨迹框架、损失函数几乎同源（关键点一致性 + 单应投影 + 网格正交保形 + 时域平滑），但把 DUT 的三个落地短板逐一补掉——**离线改在线（零未来帧）、脆弱的单一运动估计器改多检测器协同、全局双向平滑改因果动态核**，并用三线程流水线在 Jetson AGX Orin 上跑到 ~12.7 FPS。
- 一句话总结差异：**DUT 回答"无监督防抖能不能做"，NLNL 回答"无监督防抖能不能上机跑"**。若做嵌入式/无人机/在线防抖，NLNL 的架构是当前纯视觉路线里落地参考价值最高的设计之一；若做离线后期且允许逐段调参，DUT 家族画质仍有竞争力。

## 2. DUT 算法详细解读

### 2.1 背景与定位

2018 年的 StabNet 证明了 CNN 可以端到端防抖，但依赖 DeepStab 这类"手持相机 + 稳定器同步拍摄"的成对数据——采集成本高、场景覆盖窄、泛化差。DUT 的核心命题是：**经典防抖管线（运动估计 → 轨迹平滑 → 重投影）中的监督信号其实蕴含在不稳定视频自身的时空一致性里**，因此可以把管线每一步神经化后无监督训练，"只看不稳定视频就学会防抖"。

### 2.2 总体流程

```
输入帧对 ──► ①运动估计            ──► ②运动传播（稀疏→稠密网格） ──► ③轨迹平滑 ──► ④网格重投影渲染
            RFNet 关键点              多单应初始化                    3D CNN 预测
            + PWC-Net 光流            + 残差细化网络                  动态平滑核
```

**① 运动估计**：RFNet 检测 L=512 个关键点，PWC-Net 估计相邻帧稠密光流，在关键点处采样得到稀疏但可靠的运动向量。

**② 运动传播（稀疏 → 稠密网格）**，分两步：
- **多单应初始化**：对关键点运动做 K-means 聚成两类（近似前/后两个平面），每类分别用 RANSAC 估计单应 H；每个网格顶点按半径 R=200px 内邻近关键点的多数类归属选择使用哪个单应（某类占比 <20% 时并入主类）。这一步解决了单一全局单应在**视差场景**下的系统性误差——是 DUT 对传统 MeshFlow 类方法最重要的改进之一。
- **运动细化网络**：以关键点运动 + 相对距离向量为输入的 1D 卷积 + 注意力网络，为每个网格顶点预测在单应初始化基础上的**残差运动**，得到稠密网格运动场。

**③ 轨迹平滑**：把每个网格顶点跨帧串联成轨迹后，用一个 3D CNN 依据轨迹的局部时空上下文**预测逐点动态平滑核与权重 λ**，再用迭代求解器（Jacobi 式）施加平滑。与固定高斯核相比，动态核可以在快速运镜段自动"少平滑"、在抖动段"多平滑"；与传统能量优化（约 100 次迭代）相比，15 次迭代即收敛。

**④ 渲染**：平滑前后网格顶点位置差 `v̂ = v + T̂ − T` 定义每个网格单元的单应，反向 warp + 双线性插值输出，最后统一裁剪。

### 2.3 无监督损失（核心贡献）

**运动细化损失 L_MR**（权重 λ_m/λ_v/λ_s = 10/40/40）：
1. *顶点-关键点运动一致性*：网格顶点运动应与邻近关键点运动的 L1 距离小；
2. *关键点投影一致性*：由网格导出的单应应能把关键点 + 其运动正确投影；
3. *保形约束*：相邻网格边经 90° 旋转关系约束网格保持近矩形，防止扭曲。

**轨迹平滑损失 L_TS**（λ/λ_s/λ_c = 15/40/20）：
1. *时域平滑*：局部时间窗内运动差分最小化；
2. *距离保持*：平滑轨迹不能偏离原轨迹太远（控制裁剪率）；
3. *空间保形*：warp 后网格形状约束；
4. *内容保持*：关键点相对网格的位置关系保持。

> 这套"**运动一致性 + 投影一致性 + 保形 + 时域平滑**"的自监督配方成为后续所有无监督防抖工作（包括 NLNL）的模板。

### 2.4 性能与结果

- **效率**：0.63M 参数（StabNet 为 30.39M），30.42G FLOPs，480p 下 **71ms/帧**（≈14 FPS，GPU）。
- **NUS 基准**（越高越好）：

| 指标 | DUT | StabNet | DIFRINT | MeshFlow | Subspace |
|---|---|---|---|---|---|
| Stability | **0.833** | 0.790 | 0.819 | 0.810 | 0.789 |
| Distortion | **0.949** | 0.672 | 0.853 | 0.779 | 0.900 |
| Cropping | 0.704 | 0.493 | **0.862** | 0.548 | 0.639 |

- 稳定度与畸变全面领先当时的监督方法与传统方法；裁剪率低于 DIFRINT 是 warp 类方法的固有代价（DIFRINT 是插帧式全帧方法）。
- 消融验证：运动细化把单应距离从 0.069 降到 0.043、畸变分从 0.941 提到 0.956；动态核平滑 15 次迭代优于传统优化 100 次迭代（畸变 0.894 vs 0.878）；对光流噪声（5–20% 高斯/椒盐）有较好容忍度。

### 2.5 优点与局限（落地视角）

**优点**：无需成对数据；模型极小（0.63M）；管线各级可解释、可单独替换；多单应对视差场景鲁棒性明显好于单单应/纯网格方法。

**局限**：
1. **离线方法**：轨迹平滑需要完整（或长窗口双向）轨迹，无法零延迟在线运行；
2. **鲁棒性上限受制于运动估计前端**：RFNet + PWC-Net 在弱纹理、低光、红外、大动态前景下会失效，且这两个组件本身较老；
3. K-means 两平面假设与 20% 合并阈值需**逐场景调参**，复杂多平面场景可能引入畸变（作者自述部分视频需要重调阈值）；
4. warp + 裁剪范式导致裁剪率偏低（0.704），全帧需另接补全模块。

## 3. "No Labels, No Look-Ahead"（CVPR 2026）详细解读

### 3.1 动机

作者对现有方法的批评点直指落地痛点：
- 经典方法的手工关键点在弱纹理/遮挡/大运动下失效；学习方法鲁棒但不可解释、不可控；
- 固定平滑策略欠泛化，学习式平滑又易过平滑/畸变；
- **绝大多数方法（点名 DUT）是离线的**，受网络结构与全局平滑策略约束，且依赖的运动估计器不开源、复杂场景鲁棒性差；
- 现有研究几乎只覆盖"手持 + 前视 + 可见光"，无人机、红外、夜间等场景无人评测。

### 3.2 方法：经典三阶段管线 + 极轻量神经残差 + 系统级流水线

**Stage 1 运动估计——多检测器协同**：
- 融合异构检测器（传统 SIFT/ORB + 深度 SuperPoint/ALIKE）输出，加权 NMS 合并；
- **空间选择性聚类**强制关键点在网格上均匀分布（避免纹理聚集导致的传播偏差）；
- 稠密光流用 **MemFlow（因果、带记忆的在线光流）**，并在关键点邻域内用光流、邻域外用关键点插值做加权融合，得到"稀疏引导的因果流"。

**Stage 2 运动传播——EfficientMotionPro（~23K 参数）**：
- **多网格单应先验**：K-means 聚 K 类 + 逐类 RANSAC 单应 + 对网格顶点做**软融合加权**（对比 DUT 的硬多数投票归属）；
- Ghost 模块 + ECA 注意力的残差网络在单应基底上预测逐顶点残差；复杂度 O(5N·C²)，C=48。

**Stage 3 运动补偿——OnlineSmoother（~3.6K 参数）**：
- **因果核平滑器**：左填充卷积编码历史，为 x/y 方向输出可学习的 **3 抽头因果核**（有效窗口 L=7 帧），公式形如 `S_t = (λ·Σ k_r·S_{t−r} + O_t) / (1 + λ·Σ|k_r|)` ——只用过去帧，**零未来帧、零前瞻**；
- Lite LS-3D 编码器 + Star-gated 解码器。

**系统级：三守护线程流水线**（运动估计 / 传播 / 补偿各一线程，有界 FIFO 队列衔接），延迟从三阶段之和降为最慢阶段耗时。**Jetson AGX Orin 上 78.94 ms/帧 ≈ 12.7 FPS**（同平台 MeshFlow 169ms、StabNet 180ms、NNDVS 340ms）。

### 3.3 无监督损失

与 DUT 同源但因果化/自适应化：
- 传播端：关键点一致性（Charbonnier 形式）、单应投影一致性、网格正交保形——三项与 DUT 的 L_MR 一一对应；
- 平滑端：**运动自适应二阶时域惩罚**（按运动幅度衰减权重，抑制加速度）、**DFT 频域高频抑制**（因果窗内）、三角形边长/角度保持的空间畸变约束、关键点投影一致性。
- 训练：自采不稳定视频，EfficientMotionPro 100 epoch（RTX 4090 约 12 小时），OnlineSmoother batch=1 保持因果性（约 2.5 小时）。

### 3.4 UAV-Test 数据集（新基准）

92 段 / 约 26 分钟，**可见光 74 段 + 长波红外（8–14μm）18 段**，DJI M300 RTK + H20N 采集，720p/1080p @25–30FPS，覆盖城市、高速、林山、水岸、工业五类场景（3–5 级风）。首个支持**夜间/红外**评测的防抖基准。

### 3.5 结果

| 数据集 | NLNL（在线） | 最强在线基线 NNDVS | DUT（离线） | RStab（离线 SOTA） |
|---|---|---|---|---|
| NUS（C/D/S） | 0.95 / 0.98 / **0.90** | 0.92 / 0.98 / 0.87 | 0.98 / 0.88 / 0.85 | 1.00 / 0.99 / 0.94 |
| DeepStab | 0.94 / 0.91 / 0.85 | 0.93 / 0.91 / 0.84 | – | – |
| Selfie（剧烈手抖） | 0.98 / 0.93 / 0.91 | 0.97 / 0.92 / 0.91 | – | – |
| UAV-Test | 0.94 / 0.90 / **0.89** | 0.89 / 0.87 / 0.84 | – | 1.00 / 0.96 / 0.94 |

在线方法中全面第一，稳定度上**超过了离线的 DUT**，与离线 SOTA RStab 差距缩小到可接受范围（而 RStab 是重型体渲染方法，不可能在线跑）。

> 注意：不同论文对同一方法的复评分数常有出入（DUT 原文 NUS Stability 0.833，NLNL 复评为 0.85、Cropping 0.98），因为评测实现、裁剪策略与视频子集不完全一致——横向比较应以同一篇论文内部的数字为准。

**消融要点**：去掉多单应 → 抖动上升；去掉关键点协同 → 采样不均导致畸变上升；因果窗 L=7 是稳定性/保真度最优折中（L=5 更稳但失真，L=9 无增益）。

### 3.6 局限（作者自述）

极端视差或大面积动态前景下退化；红外模态依赖足够热对比度；单应先验假设平面主导运动，复杂 3D 平移场景受限；在更低端嵌入式设备上仍可能不够实时。

## 4. 逐维度对比

| 维度 | DUT（TIP 2022） | NLNL（CVPR 2026） |
|---|---|---|
| **任务设定** | 离线（需完整/长窗轨迹） | **在线、零未来帧**（严格因果，只用 I_{t−1}, I_t） |
| **延迟** | 整段视频级 | 单帧级（最慢流水线阶段 ≈ 79ms） |
| **运动估计前端** | RFNet 关键点 + PWC-Net 光流（单一来源，弱纹理/红外易失效） | SIFT/ORB/SuperPoint/ALIKE **多检测器协同** + 均匀化 + MemFlow 因果流融合 |
| **稀疏→稠密传播** | K-means 两平面多单应（硬归属）+ 残差细化网络 | K 类多单应**软融合** + Ghost/ECA 残差网络（~23K 参数） |
| **轨迹平滑** | 3D CNN 预测**双向**动态核，迭代求解（需要未来帧） | **因果 3 抽头动态核**（L=7 历史窗）+ 频域高频抑制 |
| **无监督损失** | 关键点/顶点一致性 + 单应投影 + 保形 + 时域平滑（模板首创） | 同一配方的因果化升级：+ 运动自适应二阶惩罚 + DFT 频域项 |
| **参数量** | 0.63M | **~27K（23K+3.6K）**，小两个数量级 |
| **实测速度** | 71ms/帧 @480p（桌面 GPU） | 78.94ms/帧（**Jetson AGX Orin 嵌入式**） |
| **系统工程** | 无 | 三线程流水线 + 有界队列，延迟=最慢阶段 |
| **评测场景** | NUS（手持可见光） | NUS/DeepStab/Selfie/GyRo + 自建 **UAV-Test（无人机+红外+夜间）** |
| **鲁棒性策略** | 多单应处理视差；对光流噪声有容忍 | 前端冗余（多检测器）、分布均匀化、软融合、自适应平滑——**系统性防失效设计** |
| **共同局限** | 大动态前景、极端视差下退化；warp+裁剪范式，非全帧 | 同左（单应/网格先验的共同天花板） |

## 5. 关系解读与落地启示

**继承关系**：NLNL 在框架层面完全是 DUT 范式——"经典管线骨架 + 每级神经化 + 时空一致性自监督"。它的每一项改动都精准对着 DUT 的一个落地短板：

1. DUT 离线双向平滑 → 因果动态核 + 流水线，解决**在线性**；
2. DUT 单一前端（RFNet/PWC-Net）→ 多检测器协同 + 因果光流融合，解决**前端鲁棒性**（这正是 DUT 在低光/红外/弱纹理下失效的根源）；
3. DUT 硬聚类归属 + 手调阈值 → 软融合加权，降低**逐场景调参**负担；
4. 0.63M → 27K 参数 + Jetson 实测，解决**嵌入式部署**。

**对本项目（VideoStab）的启示**：
- 若目标是**在线/嵌入式纯视觉防抖**，NLNL 的架构是当前最佳参考蓝图；即使不复现全部，"多检测器关键点协同 + 网格软融合多单应 + 因果动态核平滑"三件套可以逐个移植到现有管线；
- 若目标是**离线后期**，DUT 官方代码（PyTorch，含 RFNet/PWC-Net 权重）仍是无监督路线里最易上手的起点，但建议把前端换成现代组件（如 ALIKE/SuperPoint + RAFT/MemFlow）——这基本就是朝 NLNL 方向演化；
- 两者共同的天花板是 **2D 网格 + 单应先验**：大视差、大动态前景仍会退化，且都非全帧（需裁剪）。若这两点是硬需求，需往全帧补全（Fast Full-frame ICCV'23）或 3D 路线（RStab/GaVS）叠加，代价是失去在线性;
- 复现评测时注意：C/D/S 三指标跨论文不可直接比对，务必在自己的评测框架内统一重跑基线。
